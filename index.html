Firebase Setup Instructions

Why Firebase? Firebase provides a free real-time database that allows all 40 participants to share data instantly across all devices. No server setup required!

Step 1: Create a Firebase Project

  1. Go to Firebase Console
  2. Click "Add project"
  3. Name your project (e.g., "cop-assessment")
  4. Disable Google Analytics (optional, not needed)
  5. Click "Create project"

Step 2: Set Up Realtime Database

  1. In the Firebase Console, click "Build" → "Realtime Database"
  2. Click "Create Database"
  3. Choose location (closest to your participants)
  4. Start in "Test mode" (we'll set proper rules later)
  5. Click "Enable"

Step 3: Get Your Firebase Configuration

  1. In Firebase Console, click the gear icon → "Project settings"
  2. Scroll down to "Your apps"
  3. Click the Web icon (</>)
  4. Register your app with a nickname (e.g., "cop-web-app")
  5. Copy the firebaseConfig object

Step 4: Update This HTML File

  1. Open this HTML file in a text editor (Notepad, VS Code, etc.)
  2. Find the section that says // TODO: Replace with your Firebase config
  3. Replace the placeholder config with your actual Firebase config
  4. Save the file

Example Firebase Config:
const firebaseConfig = {
  apiKey: "AIza...",
  authDomain: "your-project.firebaseapp.com",
  databaseURL: "https://your-project.firebaseio.com",
  projectId: "your-project",
  storageBucket: "your-project.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abc123"
};

Step 5: Set Database Rules (Important for Security)

  1. Go back to "Realtime Database" in Firebase Console
  2. Click the "Rules" tab
  3. Replace the rules with the following:
{
  "rules": {
    "responses": {
      ".read": true,
      ".write": true
    }
  }
}

Note: These rules allow anyone to read/write. For production, consider adding authentication.

Step 6: Test It!

  1. Save all changes
  2. Open the HTML file in your browser
  3. Complete a test assessment
  4. Check Firebase Console → Realtime Database to see your data
  5. Open the file on another device/browser and verify data syncs

✓ Success! Once configured, all participants using this file will share data in real-time across all devices!
